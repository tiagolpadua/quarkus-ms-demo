Educational sections

Collapsed by default so the page stays concise while still surfacing architecture and stack decisions.

The content below mirrors the project documentation and examples exposed by the codebase.

The application follows a clear layered design so examples remain easy to navigate and extend.

Layer Responsibility
Resource HTTP endpoints, request/response handling, validation entry points, and OpenAPI annotations.
Service Business rules, DTO to entity orchestration, and Micrometer counters.
Persistence JPA entities, Panache repositories, named queries, native queries, and Criteria API examples.
Shared Response envelopes, pagination, logging, lifecycle hooks, UI entry point, and health checks. 
Client -> Resource -> Service -> Repository -> H2
Client -> LoggingFilter -> request correlation + metrics + traces
Client -> Shared envelopes -> ListResponse / PagedResponse
Class Purpose
LoggingFilter Logs request start and completion with correlation metadata.
ApplicationLifecycle Logs active profile during startup and shutdown.
RestEndpointLivenessHealthCheck Custom liveness check that verifies a core REST entry point.
UiHomeResource Serves this Bootstrap and Qute-powered landing page.
ListResponse<T> Canonical envelope for non-paginated lists.
PagedResponse<T> Pagination contract with items, page, and sort.

Each domain isolates its resources, DTOs, services, mappers, and persistence contracts to keep examples close to the feature they explain.

/pet

Pet domain

Create, update, delete, upload image metadata, and query pets by status or tags.

/store

Store domain

Inspect inventory and manage orders through a compact order lifecycle example.

/user

User domain

CRUD plus named query, named native query, Criteria API, and paginated list examples.

Response contract: all collection endpoints return { "items": [...] } or a paginated envelope. The API never exposes a bare JSON array.

The stack is intentionally pragmatic: official Quarkus extensions for runtime capabilities and a small set of focused libraries around mapping, formatting, and tests.

Quarkus Extensions quarkus-rest, quarkus-rest-qute, quarkus-smallrye-openapi, quarkus-hibernate-orm-panache, quarkus-smallrye-health, quarkus-micrometer, quarkus-opentelemetry, quarkus-info
Libraries MapStruct, Lombok, opentelemetry-jdbc, Bootstrap 5.3, quarkus-resteasy-problem
Maven Plugins Spotless with Google Java Format and JaCoCo coverage reporting
Testing JUnit 5, Mockito, AssertJ, WireMock, and direct validator-based tests with no Quarkus boot in the suite

The home page is rendered by Qute from templates/UiHomeResource/index.html.
